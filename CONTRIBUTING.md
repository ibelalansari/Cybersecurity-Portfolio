# Contributing

Thank you for your interest in contributing to this repository.

This repository serves as a central cybersecurity portfolio showcasing technical projects, documentation, security research, and practical laboratory work across multiple cybersecurity domains.

---

## Purpose

The purpose of this repository is to provide a structured overview of cybersecurity projects, technical documentation, and professional work while maintaining high standards of accuracy, consistency, and technical quality.

---

## Contribution Guidelines

Contributions should:

- Improve technical accuracy
- Follow cybersecurity industry best practices
- Use clear and professional language
- Include authoritative references where applicable
- Maintain consistency with the existing documentation
- Preserve the repository structure and naming conventions

---

## Repository Standards

Documentation should be:

- Well organized
- Technically accurate
- Easy to navigate
- Consistently formatted
- Based on practical experience whenever possible

---

## Naming Convention

Use descriptive names with hyphens.

### Examples

- ethical-hacking
- soc-learning-journey
- python-for-cybersecurity
- active-directory-labs
- web-security-labs

Avoid generic names such as:

- project1
- new-folder
- test
- temp
- notes

---

## Documentation Guidelines

Technical documentation should include, where appropriate:

- Overview
- Objectives
- Methodology
- Tools
- Practical Exercises
- Findings
- Mitigation
- References

---

## References

Whenever possible, reference official documentation from trusted sources such as:

- OWASP
- MITRE ATT&CK
- NIST
- Microsoft Learn
- Microsoft Security Documentation
- PortSwigger Web Security Academy
- Kali Linux Documentation
- Wireshark Documentation

---

## Code of Conduct

Please maintain respectful, professional, and constructive communication.

Constructive feedback and technical discussions are encouraged.

---

## Responsible Disclosure

Do not publish:

- Credentials
- API Keys
- Passwords
- Private Tokens
- Personally Identifiable Information (PII)
- Proprietary or confidential information

Always sanitize screenshots, logs, and reports before publishing.

---

## Disclaimer

All projects, documentation, and research included in this portfolio are conducted within authorized laboratory environments, Capture The Flag (CTF) platforms, or systems where explicit permission has been granted. Unauthorized testing of third-party systems is not supported or encouraged.
